Comandos GitHubb

Agregamos desde la web de GitHub, para usar el comando git pull y que se actualice en la pc!

  • git clone url | te trae el repositorio que creaste en la web de GitHub, en este caso "Primero"
  • pwd | me muestra el directorio donde estoy
  • ls | te muestra archivos de adentro
  • cd Primero | entra a la carpeta Primero, no movemos de directorio
  • touch comandosGit.html | crea el archivo html
  • git add comandosGit.html | agrega el archivo al repositorio para tenerlo en cuenta
  • git commit -m "mensaje" | se guarda un snapshot del archivo en ese momento, con el mensaje que le pongas, para saber que modificaciones guardaste
  • git status | no se bien para que es
  • git push | te lo sincroniza con la nube, online en el url del repositorio
  • con el git status ves si se modifico algo y no se guardo o no se commitio, figura en rojito
  • git commit -am "mensaje" | con este tenes el add y el commit 2 en 1, entonces se guarda todo lo editado para dsp pushearlo
  • y si agrego uno mas para probar bien el git -am ""
  • git reset --hard | me vuelve a la version que le indico
  • git reset --hard origin/master | me vuelve a la version que haya en GitHub
  • git log | te indica las modificaciones y quien las hizo

    Branch

  • git branch | te indica en que branch estas, si no hay ninguno va a figurar master
  • git checkout -b style | me crea un branch nuevo que se llame style y me lleva a trabajar ahi
  • git checkout main | me lleva al branch main
  • git merge styyle | me une el main con el branch styyle que hice aparte
  • en caso de que me tire error porque se modifico la misma linea en ambas versiones, tengo que fixear el error y dsp poner "git commit -am "fixed merge"" y chau
